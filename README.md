# Вывод о проделанной работе

Обучема ML модель прогнозирования стоимости автомобиля.

В первой части проведен EDA. Выполнена предобработка данных для последующего анализа. Выявлены зависомости в данных, признаки не влияющие на стоимость автомобиля. Сделны визуализации.

Во второй части работы обучена модель, использующая только ML признаки. В части опробовано обучение следующих моделей: LinearRegression, Ridge, Lasso, ElasticNet. From scratch реализована метрика R^2. При обучении моделей применялась техника кросс-валидации для определения лучших значений гиперпараметров. Реализована L0 регуляризация, но с некоторыми допущениями.

В части три к модели добавлены категориальные признаки. Проведена их оброботка для использовании в моделях. Для кодирования использовался OHE.

В четвертой части добавлены дополнительные фичи которые используются для лучшего выявления зависимостей с целевой переменной, пороговые признаки, улучшена обработка категориальных признаков, выполнена фильтрация выбросов. В результате обучена модель с r^2 ~ 0.935

В четвертой части к обученым моделям применяются бизнес метрики. Это позволять понять эфиктивность модели в бизнесе.

В пятой части созден Streamlit сервис. Приложение можно испльзовать. Оно ожидает данные в форма csv. У прилиожение есть особенность. Оно ожидает данные вместе с целевым показателем(формат учебного датасета). Все получилось удачное визуализовать. В части визуальций нужно знать, что нужно показать, а остальное дело техники. В моем случает основная проблема заключалась в том, что пришлось переность обработку датасета вручную. Еслибы я это делал не в первый раз, то код в секции EDA сразу писал под использования в streamlit. В перспективы можно добавить работу с датасетом, где нет целевого признака, добавить выгрузку предсказанных данных. Если говорить про бизнес, то нужно добавить расчет бизнес метрик.

В результате работы обученая модель показывает следующие метрики:
```
R2 test: 0.9352162567166284
MSE test: 35114106520.5398
RMSE test: 187387.58368830045
```

Метрика R2 говорит о том, что модель на ~93% описывает изменения в данных, что является хорошим показателем. MSE показывает среднее квадратичное отклоненеие. Эту метрику не очень удобно интрепретировать в рамказ задачи, поэтому дпльше используется метека RMSE. Она показывает среднее отколонение прогнозов от целевой переменной. В данном случае она говорит, что среднее отклоненение составляет 187 тысяч у.е. при медианы в 617 тысяч. Это составляет ~30%, что для бизнеса может является плохим показателем.  

Наибольший прост после использования следующих техник: скалирование данных, добавление категориальных фич, фильтрация выбросов.

Как можно было бы еще улучшить модель, но это не сделано в связи с незхваткой времени (ಥ﹏ಥ).

Для улучшения качество прогнозов не нужно было убарать признаки по их корреляции с целевой переменная так, как их можно было бы использовать для генерации новых признаков, которые могли бы быть полезны. 
Можно было бы разабраться с проблемой при обучении ElasticNet после добавления категориальных признаков. Он не успевает сойтись до необходимой точности. При этом результаты обучения были хуже чем у Ridge. 
Сделать более качественную обработку категориальных признаков. 
Улучшить нейминг переменных и организации ноутбука(многие черновые ячейки оставлены чтобы можно было понять ход моих мыслей).
Подумать над дополнитлельными способами визуализации данных и определения зависимостей в данных.
